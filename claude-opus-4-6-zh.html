介绍 Claude Opus 4.6

2026年2月5日

我们正在升级我们最智能的模型。

全新的 Claude Opus 4.6 在其前代产品的编程能力基础上进行了改进。它规划更加周密，能够持续更长时间的智能体任务，在更大的代码库中运行更加可靠，并且拥有更好的代码审查和调试能力来发现自身的错误。此外，作为我们 Opus 级别模型的首创，Opus 4.6 提供了 100 万 token 上下文窗口的测试版。

Opus 4.6 还可以将其增强的能力应用于各种日常工作任务：运行财务分析、进行研究，以及使用和创建文档、电子表格和演示文稿。在 Cowork 中，Claude 可以自主地进行多任务处理，Opus 4.6 能够代替您运用所有这些技能。

该模型在多项评估中达到了最先进的水平。例如，它在智能体编程评估 Terminal-Bench 2.0 上取得了最高分，并在 Humanity's Last Exam（一项复杂的跨学科推理测试）中领先于所有其他前沿模型。在 GDPval-AA——一项评估金融、法律和其他领域经济价值知识工作任务表现的测试1——中，Opus 4.6 以约 144 个 Elo 分的优势超过了行业次优模型（OpenAI 的 GPT-5.2）2，并以 190 分的优势超过了其前代产品（Claude Opus 4.5）。Opus 4.6 在 BrowseComp 上的表现也优于所有其他模型，该测试衡量的是模型在网上定位难以找到的信息的能力。

正如我们在详尽的系统卡片中所展示的，Opus 4.6 还展现了与行业中任何其他前沿模型一样好甚至更好的整体安全特征，在各项安全评估中表现出较低的失配行为率。

在 Claude Code 中，您现在可以组建智能体团队协同完成任务。在 API 上，Claude 可以使用上下文压缩来总结自身上下文，执行更长时间的任务而不会触及限制。我们还引入了自适应思考，模型可以根据上下文线索判断需要多大程度地使用其扩展思考能力，以及新的努力程度控制，让开发者在智能度、速度和成本之间获得更多控制权。

我们对 Claude in Excel 进行了大幅升级，并以研究预览的形式发布了 Claude in PowerPoint。这使得 Claude 在日常工作中更加得力。

Claude Opus 4.6 现已在 claude.ai、我们的 API 以及所有主要云平台上提供。如果您是开发者，请通过 Claude API 使用 claude-opus-4-6。定价保持不变，为每百万 token 输入 $5 / 输出 $25；完整详情请参阅我们的定价页面。

下文将深入介绍该模型、我们的新产品更新、我们的评估以及我们详尽的安全测试。

初步印象

我们用 Claude 构建 Claude。我们的工程师每天都使用 Claude Code 编写代码，每个新模型首先在我们自己的工作中进行测试。使用 Opus 4.6，我们发现模型无需指示就能将更多注意力集中在任务中最具挑战性的部分，快速处理较为简单的部分，以更好的判断力处理模糊问题，并在更长的会话中保持高效。

Opus 4.6 通常会进行更深入的思考，在给出答案之前更仔细地重新审视其推理过程。这在更难的问题上会产生更好的结果，但在较简单的问题上可能会增加成本和延迟。如果您发现模型在某个任务上过度思考，我们建议将努力程度从默认设置（高）调低到中等。您可以通过 /effort 参数轻松控制。

评估 Claude Opus 4.6

在智能体编程、计算机使用、工具使用、搜索和金融方面，Opus 4.6 是一个行业领先的模型，通常领先幅度很大。下表展示了 Claude Opus 4.6 与我们之前的模型以及其他行业模型在多种基准测试上的对比。

Opus 4.6 在从大量文档集合中检索相关信息方面有了很大提升。这延伸到了长上下文任务，它能在数十万个 token 中保持和追踪信息，漂移更少，并且能发现即使 Opus 4.5 也会遗漏的埋藏细节。

对 AI 模型的一个常见抱怨是"上下文衰减"，即当对话超过一定数量的 token 时性能会下降。Opus 4.6 的表现明显优于其前代产品：在 MRCR v2 的 8 针 100 万变体上——一个测试模型从大量文本中检索"隐藏"信息能力的大海捞针基准测试——Opus 4.6 得分为 76%，而 Sonnet 4.5 仅得分 18.5%。这是模型在保持峰值性能的同时能够实际使用多少上下文的质的飞跃。

总而言之，Opus 4.6 在长上下文中查找信息方面更出色，在吸收信息后的推理方面更出色，总体上具有显著提升的专家级推理能力。

最后，下面的图表展示了 Claude Opus 4.6 在各种基准测试上的表现，这些测试评估了其软件工程技能、多语言编程能力、长期一致性、网络安全能力以及生命科学知识。

安全方面的进步

这些智能提升并非以牺牲安全为代价。在我们的自动化行为审计中，Opus 4.6 表现出较低的失配行为率，如欺骗、谄媚、鼓励用户幻想以及配合滥用。总体而言，它与其前代产品 Claude Opus 4.5 一样对齐良好，后者是我们迄今为止对齐最好的前沿模型。Opus 4.6 还展示了所有近期 Claude 模型中最低的过度拒绝率——即模型未能回答良性查询的情况。

对于 Claude Opus 4.6，我们运行了所有模型中最全面的安全评估集，首次应用了许多不同的测试，并升级了我们之前使用的几项测试。我们新增了用户福祉评估，设置了更复杂的模型拒绝潜在危险请求能力测试，并更新了模型秘密执行有害行为能力的评估。我们还实验了来自可解释性（研究 AI 模型内部运作机制的科学）的新方法，以开始理解模型为什么以某些方式行事——并最终发现标准测试可能遗漏的问题。

所有能力和安全评估的详细描述可在 Claude Opus 4.6 系统卡片中找到。

我们还在 Opus 4.6 表现出特殊优势的领域应用了新的安全防护措施，这些优势可能被用于危险或有益的用途。特别是，由于该模型展示了增强的网络安全能力，我们开发了六个新的网络安全探针——检测有害响应的方法——以帮助我们追踪不同形式的潜在滥用。

我们还在加速模型的网络防御用途，利用它帮助发现和修补开源软件中的漏洞（如我们在新的网络安全博客文章中所述）。我们认为网络防御者使用像 Claude 这样的 AI 模型来帮助平衡力量对比至关重要。网络安全发展迅速，随着我们对潜在威胁了解更多，我们将不断调整和更新我们的安全防护措施；在不久的将来，我们可能会实施实时干预以阻止滥用。

产品和 API 更新

我们在 Claude、Claude Code 和 Claude 开发者平台上进行了大量更新，以让 Opus 4.6 发挥最佳表现。

Claude 开发者平台

在 API 方面，我们为开发者提供了更好的模型努力程度控制和更灵活的长时间运行智能体支持。为此，我们引入了以下功能：

  • 自适应思考。此前，开发者只能在启用或禁用扩展思考之间进行二选一。现在，通过自适应思考，Claude 可以自行决定何时更深入的推理会有所帮助。在默认努力级别（高）下，模型会在有用时使用扩展思考，但开发者可以调整努力级别使其更多或更少地选择性使用。
  • 努力程度。现在有四个努力级别可供选择：低、中、高（默认）和最大。我们鼓励开发者尝试不同的选项以找到最佳效果。
  • 上下文压缩（测试版）。长时间运行的对话和智能体任务经常触及上下文窗口。上下文压缩在对话接近可配置阈值时自动总结和替换较旧的上下文，让 Claude 能够执行更长的任务而不会触及限制。
  • 100 万 token 上下文（测试版）。Opus 4.6 是我们第一个具有 100 万 token 上下文的 Opus 级别模型。超过 20 万 token 的提示适用高级定价（每百万输入/输出 token $10/$37.50）。
  • 128k 输出 token。Opus 4.6 支持高达 128k token 的输出，让 Claude 能够完成更大输出的任务而无需将其拆分为多个请求。
  • 仅美国推理。对于需要在美国运行的工作负载，仅美国推理可用，token 定价为 1.1 倍。

产品更新

在 Claude 和 Claude Code 中，我们添加了功能，让知识工作者和开发者能够使用他们每天使用的更多工具来应对更难的任务。

我们在 Claude Code 中以研究预览的形式引入了智能体团队。您现在可以启动多个智能体作为团队并行工作并自主协调——最适合可以拆分为独立的、读取密集型工作的任务，如代码库审查。您可以使用 Shift+Up/Down 或 tmux 直接接管任何子智能体。

Claude 现在还能更好地与您已经使用的办公工具配合。Claude in Excel 以改进的性能处理长时间运行和更难的任务，可以在行动前进行规划，摄入非结构化数据并在无需指导的情况下推断正确的结构，以及在一次传递中处理多步骤更改。将其与 Claude in PowerPoint 配合使用，您可以先在 Excel 中处理和构建数据，然后在 PowerPoint 中将其可视化呈现。Claude 会读取您的布局、字体和幻灯片母版以保持品牌一致性，无论您是从模板构建还是从描述生成完整的演示文稿。Claude in PowerPoint 现已面向 Max、Team 和 Enterprise 计划以研究预览的形式提供。
