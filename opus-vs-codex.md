# Claude Opus 4.6 vs GPT-5 Codex：两家公司烧了几百亿，就为了让AI替你写代码

两家公司烧了几百亿美元，就为了一件事：让AI替你写代码。

现在，它们打起来了。

2026年2月，Anthropic发布了Claude Opus 4.6。两周前，这家公司刚发了Sonnet 4.6。而OpenAI的GPT-5和Codex编程智能体，也在过去几个月不断迭代。

两边的benchmark分数咬得死紧。

看到这个消息，人们大概分成三种反应：

**嘲笑派**："又来了，跑分大战，跟我有什么关系？"

**困惑派**："Claude是啥？Codex又是啥？它们不都是ChatGPT吗？"

**顿悟派**："等等……如果AI已经能写代码了，那程序员干嘛？"

如果你是第三种反应，恭喜你。

接下来这篇文章就是为你准备的。

---

## 先搞清楚：这两个AI到底是谁做的？

很多人一提AI就想到ChatGPT。

但AI编程这条赛道上，其实有两家公司在正面对决。

**OpenAI** —— 做ChatGPT和GPT-5的公司。Sam Altman是老板。硅谷最高调的AI公司，没有之一。

**Anthropic** —— 做Claude的公司。创始人是从OpenAI出走的兄妹Dario和Daniela Amodei。低调，但技术不低调。

两家公司的AI编程产品，各有一个杀手锏：

- OpenAI有**Codex**——一个云端编程智能体，基于codex-1（o3模型专门为写代码优化的版本）。你给它一个任务，它能在云端独立写代码、跑测试、提交PR。（来源：OpenAI官网）

- Anthropic有**Claude Code**——现在已经正式发布，集成了VS Code和JetBrains，还能通过GitHub Actions在后台跑任务。（来源：Anthropic官网）

如果你完全不懂技术，可以这样理解：

```
┌─────────────────────────────────────────────┐
│           AI编程：两种风格                    │
├──────────────────┬──────────────────────────┤
│  Claude Opus 4.6 │  GPT-5 + Codex          │
│                  │                          │
│  像资深架构师     │  像速度型全栈工程师       │
│  100万token上下文 │  74.9% SWE-bench        │
│  能连续工作数小时  │  一个提示词生成整个网站   │
│  擅长理解巨型代码库│  擅长快速生成+调试       │
│  "agent teams"   │  云端并行处理多个任务     │
│  多智能体协作     │  自动跑测试自动提PR      │
└──────────────────┴──────────────────────────┘
```

简单说：**Opus像那种看了三遍代码才动手、但一出手就是精准重构的老架构师。GPT-5 Codex像那种接到需求立马开干、速度飞快、一个prompt就能搭出整个前端的全栈选手。**

两种能力，都很恐怖。

## 跑分大战：咬得有多紧？

在AI编程领域，有一个公认的硬核测试叫**SWE-bench Verified**。

它不是那种刷题竞赛。它用的是真实的GitHub开源项目的真实issue——你得理解代码、修改代码、通过原始单元测试。500道题，全是真活。（来源：swebench.com）

来看看分数：

| 模型 | SWE-bench Verified | 其他亮点 |
|------|-------------------|---------|
| **Claude Opus 4**（2025年6月） | **72.5%** | Terminal-bench 43.2% |
| **Claude Sonnet 4** | **72.7%** | 性价比之王 |
| **GPT-5**（2025年） | **74.9%** | Aider Polyglot 88% |
| **Claude Sonnet 4.6**（2026年2月） | 据报道有提升 | ARC-AGI-2 60.4% |

（以上数据来源：Anthropic官网、OpenAI官网公开benchmark）

GPT-5在SWE-bench上以74.9%略微领先。

但Claude Opus 4在长时间持续任务上有独特优势——Anthropic官方说它能"连续工作数小时，执行数千步操作"。Rakuten用它做了一个开源项目重构，**独立跑了7个小时**。（来源：Anthropic官网Claude 4发布页）

而GPT-5的杀手锏是效率：用思考模式时，比o3**减少50-80%的token消耗**，同时性能更好。（来源：OpenAI GPT-5发布页）

这是什么概念？

同样的活，花的钱更少，干得更快。

再看OpenAI Codex：它能同时并行处理多个编程任务，每个任务在独立的云端沙箱里跑。你扔10个bug给它，它能同时修10个。（来源：OpenAI Codex发布页）

而Anthropic的反击是**Opus 4.6的"agent teams"**——多个AI智能体分工协作，各自负责一块，并行推进。（来源：TechCrunch, 2026年2月5日）

你品品，这已经不是一个AI在写代码了。

这是一个AI团队在写代码。

## 等等，这跟我有什么关系？

如果你不是程序员，你可能觉得以上都是技术圈的事。

那我告诉你一条新闻：

> **Spotify最好的程序员从2024年12月起就没写过一行代码了——全靠AI。**
>
> ——TechCrunch报道

你没看错。

不是实习生，不是初级工程师。是"best developers"——最好的开发者。

这不是什么实验项目。这是一家市值几百亿美元的上市公司，核心技术团队的真实工作方式。

而且这不只是Spotify的事。

再看一组数据：

- **GitHub 2024年开发者调查**：超过**97%**的受访者表示曾在工作中使用AI编程工具。不是"打算用"，是"已经用了"。（来源：GitHub Blog, 2024年调查）

- **Stack Overflow 2024年调查**：**76%**的开发者正在使用或计划使用AI工具。但只有**43%**信任AI输出的准确性。（来源：Stack Overflow Developer Survey 2024）

![开发者使用AI](https://images.pexels.com/photos/3861969/pexels-photo-3861969.jpeg?auto=compress&cs=tinysrgb&w=1260&h=750&dpr=2)

97%已经用过。

但只有43%信任它。

这个矛盾说明什么？

**说明大家都在用，但都在偷偷用。**

你的同事可能已经在用Claude Code写你们的核心功能了。你的下属可能用GPT-5 Codex一个小时干完了你以为要三天的活。

他们不会告诉你。

因为说出来，要么显得自己"偷懒"，要么暴露自己的工作其实没那么复杂。

## 数据冲击：这个市场有多大？

不只是开发者在焦虑。资本市场也在疯狂下注。

Anthropic最新一轮估值已经突破**600亿美元**。OpenAI的估值更是达到了天文数字级别。

两家公司烧的钱，最终都指向同一个赌注：**谁能成为开发者的默认AI助手，谁就赢了这个时代最大的生产力入口。**

GitHub Copilot（背后是OpenAI的模型）已经是最大的AI编程工具。但GitHub现在宣布，将用**Claude Sonnet 4作为GitHub Copilot新编程智能体的驱动模型**。（来源：Anthropic Claude 4发布页）

你看到了吗？

GitHub是微软的。微软是OpenAI最大的投资者。

但GitHub的Copilot，开始用竞争对手Anthropic的模型了。

这说明什么？**在AI编程这个战场上，没有永远的盟友，只有永远的性能。**

再看看这些公司的评价：

- **Cursor**（最火的AI编程编辑器）称Claude Opus 4为"编程领域的SOTA（最先进水平），在复杂代码库理解方面是一个飞跃"
- **Replit** 说Opus 4在"多文件复杂修改"上有"戏剧性的进步"
- **Cognition**（Devin AI的母公司）说"Opus 4能解决其他模型解决不了的复杂挑战"

（以上引用来源：Anthropic Claude 4发布页合作伙伴评价）

而GPT-5呢？OpenAI说它能"一个提示词就生成漂亮的响应式网站、应用和游戏"，早期测试者特别提到了它的**审美能力**——间距、字体、留白，都比以前好得多。（来源：OpenAI GPT-5发布页）

一个AI开始有审美了。

你品品。

## 真正的问题：程序员会失业吗？

不会。

但程序员这个职业，正在发生一次剧烈的**分化**。

分化成两种人：

**第一种：AI驾驶员。** 他们不再手写每一行代码。他们的核心能力变成了——理解需求、拆解架构、审查AI输出、做技术决策。他们用Claude Code和Codex当工具，效率是以前的5-10倍。

**第二种：还在手写每一行代码的人。** 不是说他们技术差。而是他们会越来越像那个坚持手洗衣服的人——不是不行，但市场不会为此买单。

Anthropic的产品负责人Scott White在接受TechCrunch采访时说了一句很有意思的话：

> "我们发现很多不是专业软件工程师的人在使用Claude Code，仅仅因为它是一个非常出色的任务执行引擎。"
>
> 使用者包括产品经理、金融分析师，以及各种行业的人。（来源：TechCrunch, 2026年2月5日）

你看到了吗？

编程不再是程序员的专属技能。

当产品经理能直接用Claude Code实现一个功能原型，当金融分析师能直接用GPT-5写数据处理脚本——

**"会写代码"这件事的稀缺性，正在急速下降。**

但"知道该写什么代码"的能力，"能判断AI写的代码是否正确"的能力——这些反而更值钱了。

这就像汽车出现后，马夫消失了，但司机出现了。后来自动驾驶出现了，司机的价值又变了——你需要的不是能开车的人，而是能设计路线、做调度、处理异常的人。

## 2026年的AI编程战场，到底谁赢了？

老实说，**平局**。

但这个平局对所有人来说都是好消息。

因为竞争意味着两家公司会疯狂迭代。2025年6月Claude Opus 4发布时，SWE-bench上还没人过50%的是新闻。8个月后，两边都逼近75%了。

mini-SWE-agent在2025年7月用100行Python代码就在SWE-bench Verified上拿到了65%。（来源：swebench.com）

100行代码。65%的通过率。

想想看，一年后会怎样？

## 最后一个问题

五年后，还需要学编程吗？

我的回答是：**需要，但学的东西完全不同。**

五年后学编程，不是学语法，不是背API，不是刷LeetCode。

而是学**如何和AI协作**、**如何审查AI的输出**、**如何做架构决策**。

就像今天你不需要学怎么造汽车，但你最好会开车。

再过几年，你可能连"开车"都不需要了——但你得知道去哪里。
