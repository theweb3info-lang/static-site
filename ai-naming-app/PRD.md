# AI取名App产品需求文档 (PRD)

## 竞品分析

### 主要竞品分析

#### 1. 起名通 (qimingtong.com)
**定位**：科学吉名开创者
**核心功能**：
- 幸福特质分析（基于积极心理学）
- 精准生辰八字计算（真太阳时）
- 五行补缺分析
- 权威汉字字库（逐字鉴定五行属性）
- 海量吉名生成
- 专家团队服务

**定价策略**：
- 免费：基础测名
- 付费：完整起名服务（预估200-800元）
- 专家服务：高端定制（预估1000+元）

**优势**：
- 与主流平台合作（腾讯、百度、360等）
- 科学化包装，降低迷信色彩
- 专业团队背景
- 用户口碑较好

**痛点**：
- 价格较高，普通用户门槛高
- 服务流程复杂，等待时间长
- 缺少移动端优化体验

#### 2. 美名腾等传统平台
**特点**：
- 基于传统五格剖析法
- 价格相对较低（50-200元）
- 功能相对简单
- 用户体验一般

#### 3. 移动端取名App
**现状**：
- 大多功能简陋，算法不准确
- 广告较多，用户体验差
- 缺乏专业背景和权威性
- 定价混乱，服务质量参差不齐

### 市场机会分析
1. **市场痛点**：专业度高的价格贵，便宜的不专业
2. **用户需求**：既要专业准确，又要价格合理，操作简便
3. **技术机会**：AI技术可以提高效率，降低人工成本
4. **移动化机会**：传统平台移动化程度低

## 核心功能模块

### 1. 八字分析模块
**输入**：姓氏 + 出生年月日时 + 性别
**处理流程**：
```
公历时间 → 农历转换 → 真太阳时校正 → 天干地支推算 → 五行强弱分析
```
**输出**：
- 八字排盘图
- 五行得分统计
- 用神分析
- 喜忌用神说明

### 2. 五行补缺推荐
**功能**：
- 分析八字中五行的旺衰
- 确定需要补强的五行元素
- 推荐对应五行属性的汉字
- 避开忌神对应的汉字

### 3. 三才五格计算
**计算项目**：
- 天格、人格、地格、外格、总格
- 基于康熙字典笔画数
- 三才配置分析（天格-人格-地格的五行关系）
- 81数理吉凶判断

**评分标准**：
- 优秀：90-100分
- 良好：80-89分
- 一般：60-79分
- 较差：60分以下

### 4. AI智能命名引擎
**生成流程**：
```
用户输入 → 约束条件设定 → AI大模型生成候选名 → 算法验证过滤 → 排序推荐
```

**约束条件**：
- 五行补缺要求
- 笔画数吉凶要求
- 音韵和谐度
- 字义积极性
- 避免生僻字、谐音不雅

**AI提示词模板**：
```
请为{姓氏}{性别}宝宝生成名字，要求：
1. 需补{缺失五行}，避开{忌神五行}
2. 人格笔画数要求：{吉利数字范围}
3. 寓意要积极向上，符合{性别}特质
4. 音韵和谐，朗朗上口
5. 不使用生僻字，避免不雅谐音
生成20个候选名字。
```

### 5. 综合评分系统
**评分维度及权重**：
- 五行匹配度：30%
- 三才五格：25%
- 字义寓意：20%
- 音韵和谐：15%
- 文化内涵：10%

**评分算法**：
```python
def calculate_name_score(name, bazi_analysis, meaning_score, phonetic_score, culture_score):
    wuxing_score = calculate_wuxing_match(name, bazi_analysis)  # 五行匹配
    sancai_score = calculate_sancai_wuge(name)  # 三才五格
    
    total_score = (
        wuxing_score * 0.3 +
        sancai_score * 0.25 +
        meaning_score * 0.2 +
        phonetic_score * 0.15 +
        culture_score * 0.1
    )
    return round(total_score, 1)
```

### 6. 诗词典故溯源
**功能实现**：
- 建立汉字-诗词典故数据库
- 为推荐名字关联相关诗词
- 展示典故来源和文化背景
- 增加名字的文化底蕴

**数据来源**：
- 《诗经》《楚辞》
- 唐诗宋词
- 经典典籍（《论语》《孟子》等）
- 成语典故

## 风险分析与应对

### 技术风险
**风险1**：八字计算准确性
- **应对**：使用多个开源库对比验证，建立测试用例

**风险2**：AI生成质量不稳定
- **应对**：设置多重过滤机制，人工审核样本

### 商业风险
**风险1**：竞品价格战
- **应对**：突出差异化价值，提升用户粘性

**风险2**：政策监管风险
- **应对**：强调文化传承，弱化迷信色彩

### 运营风险
**风险1**：用户增长乏力
- **应对**：社交裂变机制，口碑营销

**风险2**：服务质量投诉
- **应对**：建立客服体系，及时处理反馈
