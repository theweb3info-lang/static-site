# 文章自查清单（通用版）

写完文章后，逐项检查。发现问题必须修复后才能保存发布。

---

## 一、读者门槛

- [ ] 主题有没有先介绍？不能假设读者已经知道
- [ ] 有没有未解释的专业术语、英文缩写？
- [ ] 非技术读者能看懂吗？（给不懂技术的人看，能理解吗？）

## 二、事实核查

- [ ] 每个事实性陈述（人物、时间、数据、事件）是否准确？
- [ ] 数据是否使用最新版本？（不要用过时的benchmark/统计）
- [ ] 引用的名人言论是否真实？能否找到原始出处？
- [ ] 因果关系是否成立？（A发生后B发生 ≠ A导致B）
- [ ] 有没有过度引申？（一项调查不代表"所有人"，原文没说的不要加戏）
- [ ] 不确定的信息是否已删除？（宁缺勿编）
- [ ] 能否为每条关键事实指出来源？

## 三、类比审核

- [ ] 每个类比是否恰当？对应关系是否准确？
- [ ] 类比有没有过度简化导致理解偏差？
- [ ] 有没有不雅/冒犯/误导性的类比？
- [ ] 技术细节在类比中有没有被扭曲？

## 四、原创性

- [ ] 观点是否独特？有没有跟已有文章撞车？
- [ ] 是否只是搬运别人的分析框架？（搜集资料后必须提炼自己的视角）
- [ ] 有没有抄袭痕迹？（措辞、结构、论点）

## 五、内容结构

- [ ] 信息有没有重复？（同一观点出现两次以上）
- [ ] 是递进的还是平铺的？（每节应该推进认知，不是重复换说法）
- [ ] 开头能在3秒内抓住读者吗？
- [ ] 结尾有升华，不是烂尾？
- [ ] 每节都有独立价值？

## 六、文字规范

- [ ] 写作指令（"角度一""Part 1"等meta标记）是否已清除？
- [ ] 不要用 Part 1/Part 2/Part N 编号，直接用标题
- [ ] 中文语境用"总裁"不用"CEO"
- [ ] 年份/生肖是否正确？（2026年=马年丙午）
- [ ] 语气是聊天式的，不是论文式的？

## 七、格式和配图

- [ ] 字数在3000-5000之间？
- [ ] 有封面图？
- [ ] md中图片使用HTTPS全路径（不用相对路径）
- [ ] 有📊数据总结section？
- [ ] 有参考来源？
- [ ] 有互动引导（"你怎么看？"）？

## 八、平台适配

- [ ] 微信公众号描述准确？（公众号也有算法推荐，不要说"关注了才能看到"）
- [ ] 没有假设特定平台的功能/限制？表述要准确

---

**原则：宁可删掉一段不确定的内容，也不要发布一个有问题的文章。**
