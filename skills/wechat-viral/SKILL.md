# wechat-viral - 微信公众号爆文写作

基于「光纤当内存」爆文（近1w阅读，2400粉丝号）提炼的写作公式。适合将科技热点/大佬争议言论转化为高传播公众号文章。

## 触发词

- `/wechat-viral`
- "写一篇爆文"
- "公众号热文"
- "科技话题爆款"

## 核心公式：大佬疯狂言论 + 小白科普 + 思维升华

### 1. 炸裂式开头（Hook）

用"如果我告诉你……你会怎么想？"或类似反直觉句式，直接制造认知冲突。

```
如果我告诉你，[反直觉事实]，你会怎么想？
是我疯了，还是[领域]疯了？
```

紧接着引入**大佬原话**（名人背书 + 争议性），然后写三种反应：
- 嘲笑派（代入普通读者的第一反应）
- 困惑派（"我没看懂"）
- 顿悟派（"卧槽，他是认真的"）

→ 读者被勾住，想知道自己属于哪派。

### 2. 小白友好科普（Surface Story）

用**生活类比**解释技术概念，不用术语。

模板：
```
[技术概念] 就是 [生活场景]。

想象你在 [日常活动]：
- [技术A] = [生活物品A]（一句话解释）
- [技术B] = [生活物品B]（一句话解释）
- [技术C] = [生活物品C]（一句话解释）
```

经典案例：内存=灶台，硬盘=冰箱，CPU=你的手。

### 3. ASCII 图解（视觉锚点）

每篇至少 1-2 个 ASCII 框图，让读者"看到"原理。在微信里比图片加载更快，更有极客感。

```
┌──────────────────────────┐
│  标题                      │
├──────────────────────────┤
│  [组件A] ──→ [组件B]      │
│      ↑           ↓        │
│      └───────────┘        │
│  一句话总结原理             │
└──────────────────────────┘
```

### 4. "等等"转折（Twist）

当读者觉得自己懂了，抛出质疑再翻转：

```
你可能要说了：
"[读者的合理质疑]？"

这就是天才和普通人的区别。
[解释为什么表面理解是错的]
```

### 5. 数据冲击 + 成本对比

用具体数字制造"哇"感：

```
- 16GB 内存：~500 元
- 128GB 内存：~4000 元
- 1TB 内存：天价

如果能用 [替代方案]，成本能降低 90%。
```

### 6. 升华：从技术到思维

结尾不停在技术层面，而是升华到"思维方式"：

```
当所有人都在优化旧系统时，天才在重新设计新系统。
当所有人都在回答"怎么做得更好"时，天才在问"为什么要这样做"。
```

配合**历史类比**（电动车、手机上网、AI写代码），让读者感受到"这次可能也是真的"。

### 7. 互动结尾

```
💬 你怎么看？
[具体问题1]？
还是觉得 [具体问题2]？
欢迎在评论区聊聊你的想法。
```

## 标题公式

```
[人物标签] + [疯狂行为] + [情绪词]
```

示例：
- 游戏之神提了个疯狂建议：用 200 公里光纤当电脑内存，全世界都炸了
- OpenAI 前CTO说了句大实话：AI公司都在烧冤枉钱，全行业沉默了
- 马斯克深夜发了条推特，硅谷集体失眠了

关键：**人物要有辨识度**，**行为要反直觉**，**反应要情绪化**。

## 写作规则

1. **段落极短**：每段1-3句，手机阅读友好
2. **口语化**：像聊天，不像论文
3. **不假设读者有背景知识**：每个技术概念都要解释
4. **先介绍主题是什么**：不要一上来就讨论，先让读者知道在说啥
5. **信息不重复**：每个段落都要提供新信息
6. **事实核查**：数据要准确，不过度引申
7. **Benchmark用最新数据**
8. **论证严谨**：引用别人言论要有来源

## 适用场景

- 科技大佬的争议性言论/推文
- 反直觉的技术突破
- 行业颠覆性提案
- "所有人都觉得不可能"的技术方向

## 不适用场景

- 产品评测（用 /sspai-review）
- 纯技术教程（用 /jp-tech-writing）
- 职场观点文（用 /ai-paradox）
- 故事驱动的历史/文化话题（用 /liang）
