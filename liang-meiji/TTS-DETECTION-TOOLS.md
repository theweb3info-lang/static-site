# TTS断句问题检测工具评测

## 测试场景
胜海舟TTS文本（11-胜海舟-tts.txt）断句和停顿问题检测

## 测试工具
1. Copilot CLI (GitHub)
2. Codex CLI (OpenAI)
3. Qwen CLI (Alibaba)
4. Claude CLI (Anthropic)
5. 手动审查

---

## 评测结果

### 🥇 Copilot CLI
**评分：⭐⭐⭐⭐⭐ (最佳)**

**优势：**
- ✅ 精准识别"人名顿号被读出来"这个最严重问题
- ✅ 给出5个具体问题 + 改进建议
- ✅ 速度快（~10秒）
- ✅ 完全免费（GitHub Pro/Copilot订阅）
- ✅ 输出格式清晰

**命令：**
```bash
copilot -p "分析TTS断句问题..."
```

**输出质量：**
- 问题定位：100%准确
- 改进建议：具体可执行
- 格式：Markdown结构化

---

### 🥈 Codex CLI
**评分：⭐⭐⭐⭐⭐ (全面但贵)**

**优势：**
- ✅ 发现了所有5个问题（与Copilot一致）
- ✅ 分析深入，给出了SSML替代方案建议
- ✅ 准确率100%

**劣势：**
- ❌ 耗时长（~30秒）
- ❌ Token消耗高（14,561 tokens，约$0.03-0.05）
- ❌ 需要OpenAI API key

**命令：**
```bash
codex exec "分析TTS断句问题..."
```

**适用场景：**
- 需要深入分析时
- 愿意付费获得详细报告
- Copilot无法解决的复杂问题

---

### 🥉 手动审查
**评分：⭐⭐⭐⭐ (全面但耗时)**

**优势：**
- ✅ 100%准确
- ✅ 可以发现工具遗漏的边缘情况
- ✅ 免费

**劣势：**
- ❌ 耗时长（~10-15分钟）
- ❌ 需要TTS经验
- ❌ 主观判断

**适用场景：**
- 最终验证
- 工具分析后的交叉检查
- 学习TTS文本规范

---

### 🏅 Qwen CLI
**评分：⭐⭐ (不推荐)**

**优势：**
- ✅ 免费（阿里云）
- ✅ 速度快

**劣势：**
- ❌ 输出太简短，缺少具体分析
- ❌ 没有给出改进建议
- ❌ 准确率低

**命令：**
```bash
qwen -p "分析TTS断句问题..."
```

**结论：**
不适合TTS文本分析，更适合简单翻译/总结任务。

---

### ❌ Claude CLI
**评分：⭐ (太慢)**

**问题：**
- ❌ 运行时间超过45秒仍未完成
- ❌ 无输出（测试中途停止）
- ❌ 需要Anthropic API key

**结论：**
不适合快速检测，可能更适合交互式对话场景。

---

## 综合推荐

### 最佳实践（按使用频率）

#### 1. 日常快速检测
**Copilot CLI** + 手动验证
- 耗时：10秒 + 3分钟验证
- 成本：免费
- 准确率：95%+

#### 2. 深度分析
**Copilot CLI** → **Codex CLI** → 手动验证
- 耗时：10秒 + 30秒 + 5分钟
- 成本：$0.03-0.05
- 准确率：99%+

#### 3. 批量检测
写脚本调用 **Copilot CLI**，输出结构化JSON
- 自动化修复流程
- 适合处理10+篇文章

## 发现的核心问题（所有工具一致）

1. **人名顿号被读出来** ← 最严重！
2. **拼音标注被读出来**
3. **破折号造成过长停顿**
4. **英文引号被读出来**
5. **逗号堆积造成结巴感**

**修复方案：** 已集成到 `fix-tts-pauses.sh`

## 经验教训

1. **Copilot CLI是最佳选择**：快速、免费、准确
2. **Codex贵但值得**：复杂问题时使用
3. **Qwen不适合TTS分析**：仅用于简单任务
4. **Claude CLI太慢**：不适合快速检测
5. **手动验证不可省略**：工具可能遗漏边缘情况
