### 发布流程（每篇文章完成后必须执行）
1. 复制文件到 `static-site/meiji/` 对应子目录
2. **更新 index 页面**（`static-site/meiji/index.html`）— 添加新文章链接
3. **添加缓存破坏参数**（可选，更新文章时建议执行）— 运行 `python3 add-cache-param.py`
4. Git commit & push 到 GitHub Pages

### 缓存规避规则
- **问题：** 浏览器会缓存HTML和音频，导致更新后看不到新内容
- **解决：** 给资源链接添加时间戳参数（如 `?v=1234567890`）
- **自动化：** `add-cache-param.py` 批量给所有音频链接加时间戳
- **手动获取刷新链接：** `./get-fresh-url.sh <文章编号>` 生成带 `?t=timestamp` 的链接

### 中文编码规则 ⭐重要
- **字符范围：** 支持简体中文、繁体中文、日语汉字
- **编码格式：** 所有文件必须是 UTF-8 编码（无 BOM）
- **HTML声明：** 每个HTML文件必须包含 `<meta charset="UTF-8">`
- **检查工具：**
  - `python3 check-encoding.py` - 检查所有文件是否有乱码（� 替代字符）
  - `./ensure-utf8.sh` - 自动修复常见编码问题
- **常见问题：**
  - 日语地名中的汉字可能生成时出现乱码（如"本所�的" → "本所地区的"）
  - 生成文章后必须运行 `check-encoding.py` 检查
  - 如发现 � 字符，立即修复后重新生成HTML
- **环境设置：**
  ```bash
  export LANG=zh_CN.UTF-8
  export LC_ALL=zh_CN.UTF-8
  ```

## 标题规则
- 要有悬念感和吸引力，但不做标题党
- 风格：老梁式设问/反转，让人想点进去
- 可用句式：疑问句、反常识切入、人物矛盾点
- 禁止：震惊体、夸大事实、感叹号堆叠
- **标题最多两个分句**，不超过 30 字。一个反差/悬念足矣，不要塞三段话

## 音频方案
- 工具：Edge TTS（免费，无限额度）
- 声音：**zh-CN-YunjianNeural**（成熟沉稳男声）
- 命令：`edge-tts --voice zh-CN-YunjianNeural -f input.txt --write-media output.mp3`
- **人名处理（重要！）**：
  - 所有日本人名（包括全名）前后加顿号：、坂本龙马、、大久保利通、、西乡隆盛、
  - 确保TTS完整读出全名，不在姓和名之间断句
  - 例如："这位叫、大久保利通、的人" 而不是 "这位叫大久保利通的人"
  - 人名出现时必须完整读出（如"、德川庆喜、"，不能只读"德川"）
- **专有名词/词组保护**（前后加顿号防止 TTS 断句错误）：
  - 四字词语/成语：大政奉还、萨长同盟、安政大狱、尊王攘夷、废藩置县、明治维新、公武合体、版籍奉还、殖产兴业、富国强兵、文明开化、征韩论、自由民权、脱亚入欧
  - 地名：萨摩藩、长州藩、土佐藩、肥前藩、会津藩、江户城、京都御所、下关、�的�的鸟羽伏见、箱馆五稜郭、二条城、桜田门外
  - 官职/机构名：太政大臣、征夷大将军、枢密院、大老、老中、家老、参议、藩知事、奇兵队
  - 年号+年份：保持连读，如"庆应三年""安政五年""文久二年""明治元年"前后加顿号
  - 书名/作品名：海国图志、劝学篇、武士道、西洋事情、自助论
  - 战役/事件名：鸟羽伏见之战、戊辰战争、西南战争、佐贺之乱、禁门之变、池田屋事件、樱田门外之变、黑船来航、萨英战争、下关战争、神风连之乱
  - 数字+量词：整体保护，如"两万石""三百藩士""一万五千人"
  - 其他高频词组：海军传习所、松下村塾、萨摩学生、岩仓使节团、船中八策、遣外使节
- **TTS 最佳参数**（经测试确定）：
  - 声音：zh-CN-YunjianNeural（成熟沉稳男声）
  - 语速：`--rate=+3%`（稍快但不急促）
  - 音调：`--pitch=-8Hz`（更低沉有磁性）
  - 命令示例：`edge-tts --voice zh-CN-YunjianNeural --rate=+3% --pitch=-8Hz -f input.txt --write-media output.mp3`
- 音频文本需单独处理（生成独立的 -tts.txt 文件，不修改原始 md）：
  1. 从 Markdown 清理格式，生成初始 -tts.txt
  2. 章节过渡处理（评书风格过渡句）
  3. **运行 `preprocess-tts.sh <file>`** 统一处理：
     - 多音字校正（还→环，年号保护，专有名词保护）
     - 儿化音书面化（自个儿→自己，事儿→事情等）
     - 日本人名保护（前后加顿号）
     - 括号别名清理（删除括号内容）
     - 清理重复顿号
  4. 生成音频：`edge-tts --voice zh-CN-YunjianNeural --rate=+3% --pitch=-8Hz -f <tts-file> --write-media <output.mp3>`
- **章节过渡处理（关键！）**：TTS 文本中不要直接读出"一、XXX"这种章节标题，要替换成评书/说书风格的过渡语，让听感丝滑自然。常用过渡句式：
  - 开篇引入："话说……" / "要说这事儿，还得从……说起"
  - 场景转换："花开两朵，各表一枝" / "咱们先按下这头不表，单说……"
  - 时间推进："这事儿还没完" / "转过年来……" / "且说几年之后……"
  - 话题转折："说到这儿，您可能要问了……" / "可事情哪有这么简单"
  - 高潮铺垫："接下来这一段，才是真正精彩的地方" / "好，关键时刻到了"
  - 收尾回顾："说到最后……" / "咱们回过头来看……"
  - 通用衔接："且说" / "再说" / "却说" / "单说" / "要说"
  - 根据上下文语境自然选择，不要机械套用，每篇每个过渡都不一样
- 多音字处理：对容易读错的多音字，在 TTS 文本中用注音或同音字替换
  - 例：大隈重(zhòng)信 → 确保正确读音
  - 人名中的特殊读音要特别注意（如：勝海舟的"勝"读 shèng）
  - 还：大政奉还(huán)、还于朝廷(huán) → TTS文本替换为"大政奉环""环于朝廷"
  - 生成前人工检查一遍多音字列表
- **别名/改名处理**：TTS文本中删除所有括号及括号内的别名，保持简洁。例如：
  - 原文："长州的木户孝允（桂小五郎）"
  - TTS文本："长州的、木户孝允、"
  - 避免语音啰嗦，读者在原文中已看到别名信息
  - 常见案例：木户孝允（桂小五郎）、大村益次郎（村田蔵六）、伊藤博文（伊藤俊辅）等
