🔥 这不是又一个图表库，这是BI的重新定义

先说清楚Shaper到底是什么，免得你以为这又是一个"开源概念"。

传统的BI工具就像装修公司——你得学会用它们复杂的界面，拖拖拽拽半天，调各种参数，最后还不一定满意。而Shaper更像是你亲自动手：你会写SQL，你就会做图表。

想象一下：

Tableau：『拖拽字段到行列，选择图表类型，设置聚合方式，调整颜色方案...』（10分钟后）

Shaper：『SELECT date_trunc('week', created_at)::XAXIS, category::CATEGORY, count()::BARCHART_STACKED FROM dataset GROUP BY ALL』（10秒搞定）

区别看出来了吗？一个是"学会工具"，一个是"用你已知的技能"。

Shaper的核心理念就是：为什么要为了做图表学一套新语法，当你已经会SQL了？

SELECT 'Sessions per Week'::LABEL;
SELECT
  date_trunc('week', created_at)::XAXIS,
  category::CATEGORY,
  count()::BARCHART_STACKED,
FROM dataset
GROUP BY ALL ORDER BY ALL;

写完这几行SQL，你就得到了一个完整的堆叠柱状图，包含标题、时间轴、分类统计。图表立即生成，还能直接嵌入到网页或应用中。

🎯 DuckDB加持，性能碾压传统方案

Shaper最聪明的地方，就是选择了DuckDB作为数据引擎。

DuckDB是什么？简单说就是"数据分析界的SQLite"——轻量、快速、无需配置。但性能上，它在很多场景下能吊打传统的数据仓库。

举个例子：

  • 传统BI方案：数据存在MySQL → ETL工具处理 → 导入BI工具 → 制作图表 → 部署到服务器
  • Shaper方案：docker run taleshape/shaper → 写SQL → 图表就绪

时间对比：

  • Tableau + SQL Server：环境搭建3天，数据建模2天，图表制作1天，部署配置1天 = 1周
  • Shaper + DuckDB：docker run命令30秒，写SQL查询5分钟，图表生成1秒 = 5分钟

更夸张的是，Shaper支持直接查询几乎任何数据源：

-- 直接查询S3上的CSV文件
SELECT * FROM 's3://my-bucket/data.csv';

-- 查询Google Sheets
SELECT * FROM 'https://docs.google.com/spreadsheets/...';

-- 查询远程Postgres
SELECT * FROM postgres_scan('host=... dbname=...');

不需要导入，不需要ETL，直接查询，直接出图。

🚀 不只是图表，这是完整的数据平台

Shaper最被低估的地方，就是很多人把它当成"又一个图表工具"。但它实际上是一个完整的数据平台。

数据摄取API：

curl -X POST http://localhost:5454/api/v1/data/events \
  -H "Content-Type: application/json" \
  -d '{"user_id": 123, "action": "click", "timestamp": "2026-02-19"}'

实时数据进来，图表立即更新。不需要复杂的流处理框架，不需要Kafka集群。

嵌入式分析：

// 嵌入到React应用中
import { ShaperDashboard } from '@taleshape/shaper-react';

<ShaperDashboard 
  dashboardId="your-dashboard-id"
  token="jwt-token-for-security"
/>

自动化报告：

  • 定时生成PDF报告并邮件发送
  • Excel导出支持复杂格式
  • 密码保护的分享链接

Git版本控制：

dashboards/

⚡ 开发者友好：5分钟上手体验

让我们实际体验一下Shaper到底有多简单：

步骤1：启动Shaper（30秒）

docker run --rm -it -p5454:5454 taleshape/shaper

步骤2：打开浏览器访问 http://localhost:5454/new

步骤3：写一个简单的查询

SELECT 'Sales Performance'::LABEL;

SELECT 
  'January'::MONTH, 100000::VALUE, 'Product A'::SERIES UNION ALL
SELECT 'February', 150000, 'Product A' UNION ALL  
SELECT 'March', 120000, 'Product A' UNION ALL
SELECT 'January', 80000, 'Product B' UNION ALL
SELECT 'February', 90000, 'Product B' UNION ALL
SELECT 'March', 110000, 'Product B';

SELECT 
  MONTH::XAXIS, 
  SERIES::CATEGORY, 
  VALUE::LINECHART
FROM sales_data;

3分钟后，你就有了一个专业的销售趋势图表，还可以直接嵌入到你的应用中。

对比一下传统方案：

  • Tableau：下载安装包（1GB+），安装配置（30分钟），学习界面（2小时），制作同样图表（30分钟）
  • Power BI：购买许可证（$10/月），下载安装，连接数据源，学习DAX语法...

差距是不是很明显？

🌊 这是数据民主化的开始

Shaper代表的不只是技术进步，更是数据分析能力的民主化。

过去，数据分析是这样的：

  • 业务人员：有需求，不会工具
  • 数据分析师：会工具，不懂业务
  • IT运维：管服务器，对分析内容一知半解

结果就是：一个简单的报表需求，要在3个团队之间来回沟通，最终耗时几周甚至几个月。

Shaper改变了这个局面：

  • 会SQL的开发者：直接写查询，立即看结果
  • 不会SQL的业务人员：让AI生成SQL，自己调整参数
  • 数据分析师：专注于复杂建模，不用纠结工具使用

最重要的是，开源意味着没有供应商锁定。你的分析逻辑都是标准SQL，随时可以迁移到任何其他平台。不会像使用商业BI工具那样，一旦深度依赖就很难摆脱。
